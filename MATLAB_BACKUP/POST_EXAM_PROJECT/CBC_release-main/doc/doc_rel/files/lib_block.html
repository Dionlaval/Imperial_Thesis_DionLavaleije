Library Blocks

The software is distributed as a library, ensuring seamless integration and ease of use within Simulink. Upon executing setup_cbc.m, all necessary protected blocks are extracted, and the contents of the src folder are automatically added to the MATLAB path. This process includes both protected and unprotected blocks provided with the software. Additionally, the library folder, containing the core library, is also added to the MATLAB path.

Accessing the Library

Once the setup process is complete, users can create a new Simulink model and access the library through the Simulink Library Browser. The library is listed under CBC Library. If the library does not appear immediately, try refreshing the Library Browser by right-clicking on the library list and selecting Refresh Library Browser.

After opening the library, users can drag and drop the required blocks into their Simulink model to construct their system.

Protected Blocks and Associated Data

The protected blocks included in the software are distributed as MATLAB project files (.mlproj). When setup_cbc.m is run, these files are extracted into designated folders. Certain blocks also require additional data stored in .mat files, which will be located in their respective extracted folders.

Users must manually load the corresponding .mat file when using these blocks, as there is currently no automated mechanism for this. If an error occurs during simulation regarding missing variables, it is likely due to an unloaded .mat file. Ensure that the required file is manually loaded before using the block.

Alternative Method: Manual Block Access

While using the library is the recommended approach, users also have the option to manually locate and copy blocks from the src folder. Protected blocks can be accessed from their harness files, which are stored in their respective extracted folders and follow the naming convention: 

*_harness.slx

Unprotected blocks can be accessed directly from the src folder and copied into the model without additional steps.

Although this manual method is available, the library is designed to streamline workflow and simplify block integration, making it the preferred method for using the software.
